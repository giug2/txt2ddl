The data model defines a system for managing tax forms, divided into five entities: Taxpayer, Bank, Tax, Form and Line. The Taxpayer entity contains the personal data of the tax subject, uniquely identified by the Tax Code, which acts as the primary key (PK), and includes the Name and City attributes. The Bank entity catalogs credit institutions, using a Code such as PK and a BankName attribute. The Tax entity defines the types of tax, each identified by a Code (PK) and described by a TaxName. The Form entity represents the single declaration, identified by an ID (PK) and containing a Total value. Each Module is associated with a single Payer and a single Bank, establishing two one-to-many relationships. Finally, the Row entity describes the individual payment items that make up a form, with its own ID (PK), Amount and Year. Each Row belongs to a single Module and refers to a specific Tax, implementing two further one-to-many relationships that complete the model structure.
