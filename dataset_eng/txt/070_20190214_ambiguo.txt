In practice, at the center of everything is Shipping, which is the main concept, identified by its code and that's it. Every shipment is connected to a lot of other stuff. For example, there is always a Sender who sends it, a person with his own code and name; the same sender can obviously make a bunch of different shipments, but each shipment comes from just one guy. Then, this shipment must go somewhere, that is, to a Destination, which has its own code and description. Many shipments can converge towards a destination, but a shipment only goes to one place. These destinations, in turn, are grouped into a kind of more general Category, so a category can contain several destinations, but a destination only belongs to one category. There is also the Shipment Type, which defines its nature, and here too, a type can be applied to a lot of shipments, but each shipment is only of one type. The shipment itself is made up of one or more Parcels; therefore, various packages are linked to a shipment, but a specific package is only part of that shipment, so much so that to identify it you need its number and the code of the shipment to which it is attached. Finally, each package has its own size, such as small, medium, large. One size can be used for an infinite number of packages, but each package has its own specific size.
