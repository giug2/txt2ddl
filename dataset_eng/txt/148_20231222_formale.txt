The data model describes a logistics system for managing shipments. The geographical structure is defined by the entities REGION, identified by a CODE, and PROVINCE, identified by an ABBREVIATION; each region contains one or more provinces. Multiple WAREHOUSES can be located within each province, each with a unique CODE and ADDRESS. Operations are managed by COURIER, identified by an ID and a COMPANY_REASON, with each courier associated with a single region of competence.

A ROUTE represents a shipping route and is identified by a primary key made up of its CODE and the identification of the COURIER that manages it. Each route connects a Departure and Arrival warehouse and is associated with a specific VEHICLE TYPE, defined by a CODE and a DESCRIPTION. Operational planning is managed by the PLANNED SERVICE entity, which represents the association between a ROUTE and an OPERATIONAL DAY (identified by a NUMBER). This service, identified by the composite key of its parent entities, defines the SCHEDULED_ARRIVAL_TIME and the SCHEDULED_DEPARTURE_TIME.

The actual transportation execution is tracked in the ACTUAL SHIPMENT entity, which represents an instance of a SCHEDULED SERVICE on a specific DATE. This entity, whose primary key is composed of the date and the reference to the service, records the REAL_DEPARTURE and REAL_ARRIVAL times. Each actual shipment is associated with a specific VEHICLE, identified by its PLATE. Each vehicle belongs to a VEHICLE TYPE, establishing a link between planning and the resources actually used.
