Basically, here we're talking about concerts and everything that revolves around them. There is the **Concert**, which is the set of pieces played, with its own title and two lines of description to make you understand what it is like. Each concert is linked to a single **Orchestra**, which has a name, a director and is somewhere. Clearly, the same orchestra can do a lot of different concerts. A concert program can be repeated several times, so we have **Events**, which are basically the dates and times when that concert is performed. Each event is held in only one **Theatre**, which is identified by a code and has a name, but a theater can obviously host many events. Inside a concert there are several musical **Pieces**, and there is also information on in which order they are played. A piece can be included in many concerts and a concert has many pieces, which is very flexible. Each piece, then, has its **Composer**, and a composer can have written an infinite amount of stuff. Now, as regards who plays, the matter is a bit complex: the orchestra is made up of a series of "posts" or roles, which here we call **Composition**. Each "composition" is linked to a single **Musician**, who has his serial number, name and surname, and to a specific **Instrument**. So, essentially, this "composition" tells us that that musician, in that orchestra, plays that instrument. A musician can be associated with more than one of these "compositions", while each "composition" is specific to only one musician and refers to only one type of instrument.
