Basically, we have a few things to keep an eye on. There is a list of all the projects, and each one has its own unique code to avoid confusion. Then, obviously, there is the list of employees, and each is identified by its serial number. There is also a sort of job catalogue, where each type of job has its own code and a short description that explains what it is. At the heart of it all is a table that we could call assignments, which serves to connect people to projects. In this table, to create an entry, you put together an employee's identification number and a project code, and that pair cannot be repeated. Each project can have a lot of these assignments linked, and the same goes for an employee, who can be assigned to multiple things. In some way, tasks are also linked to these assignments; it seems that each type of task can be associated with quite a few different assignments, even if it is not exactly explicit how this link is recorded directly in the assignment itself.
