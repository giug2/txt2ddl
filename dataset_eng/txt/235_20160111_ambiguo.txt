So, in practice we have a series of packages or subscription plans, each identified by its own unique code to avoid confusion. At the same time, there is a list of all the possible features, such as the extra options that can be had, and these also have a code to be recognized on the fly and a name to understand what it is. The great thing is that these two things are connected in a fairly flexible way: any one plan can include a mix of different features, and, at the same time, a single feature can be present in multiple different packages. The interesting thing is that when a certain functionality is combined with a certain plan, a certain monthly cost is associated with this specific combination, which is therefore not fixed for the functionality itself, but depends precisely on the combination created with the chosen plan.
