Show, in the box on the following page, a conceptual scheme for a reality relating to the management of a set of projects of a company, according to the following specifications (much simplified compared to reality, but with some details that must be specified as required).
• Each project has a code, name and operational headquarters; the operational offices are "standardised", in the sense that the database contains all the possible company offices, each with street, house number, city (with code and name) and region (with code and name).
• For each project, the team members are represented, with serial number, name, surname, date of birth, percentage of allocation on the project (for example Mario Rossi can be allocated 30% on the 'Alfa' project) and any hierarchical relationships existing between the members (even from different projects), with the type of relationship (for example Mario Rossi is manager of Aldo Verdi and collaborates with Anna Pini). The relationships are represented with code and description.
• Each project has a work plan, composed of a set of tasks to be performed. Tasks are characterized by a code (unique per task for all projects) and a name. Each task has an estimated budget, which varies and depends on the project, while the task name is identical for all projects.
• Each task requires a set of resources, which are each described by a code and a name (unique for all projects).
• We want to keep track of possible project risks, indicating for each resource which risks it is associated with. Please note, everything is very simplified and you only want to represent the risks (with code and name) and associate them with the resources.
