So, in practice, at the center of everything are the Projects, each with its own identification code to avoid confusion. Each project is always linked to only one Client, but a client, of course, can have a lot of different projects at stake. These customers, in turn, are not placed at random, but are grouped into categories, let's say Sectors, each with its own code and description; every customer necessarily belongs to one of these sectors. Returning to projects, each of them always has a status, such as "in progress" or "finished", which is drawn from a list of possible states. Likewise, each project is managed by a certain department of the company, which has its own name and code, and a department can easily follow multiple projects at the same time. Then, within each project, there are the various things to do, the Tasks. A task has its own number, but it only makes sense within the project to which it is linked, in fact it is closely linked to that. A project can be broken up into an infinite number of tasks, but a single task always belongs only to one project. Finally, a Priority label is attached to each task, which is chosen from a specific list where each priority has its own code and description.
