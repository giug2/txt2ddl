The data model defines a system for managing conferences, divided into five main entities. The Country entity, identified by a Unique Code (PK) and a name attribute, has a one-to-many relationship with the City entity. Each City, defined by a name, belongs to a single nation and is identified by a primary key composed of its own Code and the foreign key fk_nation. At the same time, the Area entity, with Code (PK) and name, groups together multiple Matter entities in a one-to-many relationship; each Matter, in turn, is identified by a Code (PK) and a name. The central entity Conference, identified by a Code (PK) and described by a Title, is the point of convergence of the model. There is a one-to-many relationship between City and Conference, indicating that a conference takes place in a single city, which can host multiple conferences. Similarly, a one-to-many relationship links Subject and Conference, establishing that each conference deals with a single subject, while a subject can be the subject of multiple conferences.
