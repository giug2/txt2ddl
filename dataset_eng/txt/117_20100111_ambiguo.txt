So, basically here we're talking about concerts and all the stuff that goes around them. There is the concert, which is the main event, identified by a code, with its own title and a description to make it clear what it is about. Each concert, of course, is played by a specific orchestra, which has a name and a conductor. An orchestra can do a lot of different concerts, but a concert is always that one. Inside the orchestra there are the musicians, each with their serial number, name, surname and the instrument they play. It's a bit flexible, in the sense that an orchestral player can play in multiple orchestras and an orchestra is made up of a lot of people.

A concert is then made up of various musical pieces, each with its own code and title. The nice thing is that a piece can be played in many different concerts, and a concert is made up of many pieces, so there's a nice mix. Each piece has its own author, a musician, who also has a code and a name. A musician can write a lot of stuff, but each piece is linked to only one author.

Since a concert can be a success, there are repeats. Each rerun is basically a date on which a certain concert is redone, so each rerun is tied to one and only one concert, but a concert can have a lot of reruns. Finally, each replica is held somewhere, in a room, which has its own code, a name and a capacity to know how many people are there. Clearly in a hall there can be a lot of reruns of different concerts over time.
