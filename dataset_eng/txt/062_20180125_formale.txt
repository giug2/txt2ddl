The data model defines the structure for managing conference events. The Series entity, identified by a Code (PK) and a Title, groups together multiple events. Each Congress belongs to a single series, establishing a one-to-many relationship, and is identified by a primary key composed of Code and fk_series, also possessing a Data attribute. Each conference is held in a single City (relationship N:1), an entity defined by Code (PK) and Name. A congress is divided into several sessions (1:N relationship); the Session entity is identified by a Code (PK) and described by Title and Time. Participation in events is modeled by a many-to-many relationship between Session and the Person entity with affiliation. The latter, identified by a Code (PK), acts as an associative entity, connecting a Person (identified by PK Code and Name) and an Institution (identified by PK Code and Name) through two distinct many-to-one relationships. Finally, each Institution is based in a single City, defining a further many-to-one relationship.
