So, basically there's a whole structure to handle some of the corporate stuff. At the highest level we have the Companies, each with its own name and code to recognize it. Then, each company is divided into various Divisions, such as departments. Each division has its own specific name and, obviously, is connected to only one company, it cannot be in the middle. To be sure which division we are talking about, its ID is not enough, but you also need to know which company it belongs to.

Inside these divisions there are people, Employees. We know everyone's name and surname, and each has its own identification code, but this code is unique only within its division. So, to find an employee, you need to know both their ID and what division they work in. Essentially, each person is assigned to one and only one division.

Always linked to the divisions, there are the various activities, the Projects. Each project has its own name and code, but even in this case the code only makes sense within the division that manages it. A project, therefore, belongs to a specific division and that's it. The slightly more intertwined part is that employees and projects are connected to each other in a flexible way: one employee can work on many different projects at the same time, and, conversely, a single project can have a lot of people working on it.
