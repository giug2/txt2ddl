The data model defines the organizational structure through four main entities: HEADQUARTERS, DEPARTMENT, MANAGER and ASSISTANT. The SEDE entity is uniquely identified by the `sede_id` attribute. The DEPARTMENT entity, identified by `pk_department`, is linked to the LOCATION entity by a one-to-many (1:N) relationship, whereby each location can contain multiple departments, but each department belongs to only one location. The MANAGER and ASSISTANT entities, identified by `pk_manager` and `pk_assistant` respectively, are both associated with a DEPARTMENT via a many-to-one relationship (N:1); this implies that each manager and each assistant are assigned to a single department, whereas a department can include multiple managers and assistants. There is also a many-to-many (N:N) relationship between MANAGER and ASSISTANT, which establishes that a manager can be supported by multiple assistants and an assistant can serve multiple managers.
