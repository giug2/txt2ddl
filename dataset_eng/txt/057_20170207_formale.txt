The data model defines four main entities: Client, Department, Project and Employee. The `Client` entity is uniquely identified by the `client_id` primary key. The `Department` entity is identified by the primary key `department_id`. The `Project` entity is defined by a composite primary key made up of `project_id` and `fk_client`; this last attribute acts as a foreign key (FK) towards the `Client` entity, establishing an N:1 cardinality relationship in which a client can have multiple projects, but each project is associated with a single client. Similarly, the `Employee` entity has a composite primary key composed of `employee_id` and `fk_department`. The `fk_department` attribute is a foreign key that references the `Department` entity, implementing an N:1 relationship whereby a department can include multiple employees, while each employee belongs to only one department. Finally, there is an N:N cardinality relationship between the entities `Employee` and `Project`, indicating that an employee can be assigned to multiple projects and, conversely, a project can involve multiple employees.
