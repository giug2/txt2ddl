So, in this system there is some stuff that revolves around travel and study. First of all, we have airports, each with its own identification code and name, just to clarify. Then there are the cities of departure, also with a code and a name. The somewhat confusing thing is that an airport can be connected to a lot of cities and, vice versa, a city can have to deal with multiple airports, it's a bit of an open relationship.

Each airport, however, is physically located in only one location, which in turn is identified by a code and a name. A locality, if it is large, can easily host more than one airport. Other things happen in these locations: for example, tests can be done. Each exam has its own code, name and cost. A single location may offer the opportunity to take different types of exams.

Always in the localities we find the colleges. A locality may have several colleges, and we know a few things about each of these: a code, the name, where it is located with the address, a telephone number and some kind of value in hours. Each college then offers various accommodation solutions, each defined by a specific period that identifies it and with its own price. Finally, there is the matter of reductions. There are different types of discounts, each with a code and description, and the connection with colleges is flexible: one college can offer many different discounts and, at the same time, the same type of discount can be applied to a bunch of different colleges.
