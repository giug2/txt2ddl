The system manages a registry of the Departments, each uniquely identified by a Code and described by Name, Location and Contact. At the same time, a catalog of Study Courses is maintained, each defined by a Code, Name, Year of Foundation and a reference to the Coordinator. The latter is linked to the Coordinators entity, which collects information on managers, specifying their identification ID, surname, name and start and end dates of the assignment. To establish the training offer, the Teaching Provided report associates a Department Code with a Course Code, also indicating the Academic Year in which the course is active in that specific department. The teaching staff of each course is managed through the Teachers entity, identified by a Serial Number, Surname and Name. Their assignment to a specific course is traced in the teaching relationship, which connects a MatricolaTecente to a specific course provided via the DepartmentCode and CourseCode pair. Administrative management is based on an income classification defined in Income Bands, where each type has a Code and a Description. The amount of contributions is established by the Contribution Quota entity, which sets an Amount for each Income Range Code in relation to a specific Department Code. Finally, the system provides a concession mechanism through the Exemption Types, cataloged with Code and Description. The actual application of these concessions is defined in the Applied Exemptions table, which assigns an exemption percentage to an Exemption Type Code for a specific Department Code, thus allowing the same concession to have different values ​​depending on the department that applies it.
