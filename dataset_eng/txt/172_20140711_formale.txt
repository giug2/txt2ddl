The data model defines four main entities: Warehouse, Employee, Lot and Product. The Warehouse entity is uniquely identified by a Warehouse Code (PK) and has an Address attribute. The Employee entity is characterized by a Serial Number (PK) and a Name. The Lotto entity is defined by a Lotto Code (PK) and a Production Data. Finally, the Product entity is identified by a SKU (PK) and includes a ProductName. The relationships between these entities are structured as follows: there is a one-to-many (1:N) relationship between Warehouse and Employee, where each warehouse can be associated with multiple employees, but each employee belongs to a single warehouse. A one-to-many (1:N) relationship is also established between Employee and Lot, indicating that an employee can manage multiple lots, while each lot is managed by a single employee. Finally, a one-to-many (1:N) relationship connects Product and Lot, specifying that a product can appear in multiple lots, but each lot corresponds to a single product.
