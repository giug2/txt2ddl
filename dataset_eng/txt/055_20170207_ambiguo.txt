In practice, there is a system that revolves around people, the "Taxpayers", who are identified by their tax code and whose name and city we also know. Each of these people is linked to a lot of "Forms", which are documents with their own ID and total. These forms are not only related to people, but also to banks; in fact, each "Bank", which has its own code and name, seems to manage quite a few of these modules. If we look inside a single "Module", we discover that it is made up of many "Rows", so for each module there are multiple lines. Each "Row" has its own identifier, an amount and a reference year. Finally, each row is linked to a specific type of "Tax", which in turn has a code and a name. This means that the same tax can appear on lots of different lines, scattered across all the forms.
