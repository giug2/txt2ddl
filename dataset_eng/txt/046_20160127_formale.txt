The data model defines three main entities: `Store`, `Brand` and `Category`. The `Store` entity represents a store, uniquely identified by a `Code` attribute that acts as the primary key (PK), and further described by `Name`, `Address` and `Floor`. The `Brand` entity is identified by a `Code` (PK) and has a `Brand Name`. The `Category` entity classifies products, using a `Code` as the primary key (PK) and a descriptive `Category Name`. The relationships between these entities are hierarchical. There is a one-to-many (1:N) relationship between `Category` and `Brand`, indicating that each category can contain multiple brands, but each brand belongs to only one category; this link implies a foreign key (FK) in the `Brand` entity that references the `Category`. Similarly, a one-to-many (1:N) relationship links `Brand` and `Store`, establishing that a brand can be distributed across multiple stores, while each store is associated with a single brand, implemented via a foreign key in `Store` that points to the reference `Brand`.
