So basically here we have some stuff to manage concerts. Each concert has its own code, a title and perhaps a description to understand what it is about. Obviously, a concert is held in a certain theatre, which also has its own code and name, and is played by a specific orchestra, which we recognize by its name and which has its own venue. Then, a concert program can be done on multiple dates, so each event is basically a specific date and time linked to that concert.

Speaking of music, the concerts are made up of various pieces, each with its own code and title. The interesting thing is that a piece can end up in a lot of different concerts, and a concert is a sequence of multiple pieces; in fact, the position of each piece in the concert lineup is also marked. Each piece, naturally, has its composer, who we identify with a code and whose name and surname we know; a composer obviously can have written a lot of pieces.

Finally, there is the orchestra. The orchestra is made up of musicians, each with their own number, name and surname. To know who plays what and in which orchestra, there is a kind of register, a table called 'Composition'. Each line of this register, which has its own unique code, is a link that tells us that a certain musician, using a certain instrument (which in turn has a code, a name and a type), is part of a given orchestra. In short, it is the way to keep track of the orchestra's formation, piece by piece, musician by musician.
