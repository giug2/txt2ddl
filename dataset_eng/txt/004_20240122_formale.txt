The data model defines four main entities: MANAGER, DEPARTMENT, ASSISTANT and HEADQUARTERS, each uniquely identified by a primary key respectively called pk_manager, pk_department, pk_assistant and pk_sede. The relationships between these entities are structured as follows: the LOCATION entity is linked to the DEPARTMENT entity by a one-to-many (1:N) relationship, indicating that each department belongs to only one location, while a location can host multiple departments. There are also three many-to-many (N:N) relationships. The first connects MANAGER and DEPARTMENT, so a manager can manage multiple departments and a department can be managed by multiple managers. The second N:N relationship exists between MANAGER and ASSISTANT, establishing that a manager can have multiple assistants and an assistant can support multiple managers. Finally, a many-to-many relationship is defined between ASSISTANT and DEPARTMENT, implying that an assistant can be assigned to multiple departments and, vice versa, a department can employ multiple assistants.
