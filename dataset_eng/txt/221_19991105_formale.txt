The data model describes a project management system structured around five fundamental entities: `Employee`, `Project`, `TaskType`, `Task` and `Document`. Each entity has a unique identifier as its primary key (PK): `employee_id` for `Employee`, `project_id` for `Project`, `document_id` for `Document`, and `tasktype_id` for `TaskType`. The `Task` entity has a composite primary key, consisting of the `task_id`, `fk_tasktype` and `fk_project` attributes, with the latter two also acting as foreign keys (FK). The interconnections between entities are defined by the following relationships: a one-to-many (1:N) relationship links `TaskType` to `Task`, establishing that each task must belong to a single typology, while a typology can categorize multiple tasks. A many-to-one (N:1) relationship links `Task` to `Project`, indicating that each task is part of a single project, which in turn comprises a set of tasks. There is a many-to-many (N:N) relationship between `Employee` and `Task`, which allows an employee to be assigned to multiple tasks and a task to be handled by multiple employees. Finally, a many-to-one (N:1) relationship between `Task` and `Document` specifies that multiple tasks can be associated with a single document.
