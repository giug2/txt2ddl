So, we basically have a system for managing all the university stuff. There are **Students**, each with their own `Matriculation number` which is like a university tax code, and then the usual data such as name, surname and telephone number. Each student is linked to a **Study Plan**, which is practically the list of exams they must take; the strange thing is that one student can have multiple plans and one plan can be followed by multiple students, a bit of a entanglement. Each floor has its own code, `Plan_ID`, to avoid confusion.

This `Study Plan` is made up of many lines, and each line is a **Course in the Plan**. So, a plan is a container for a lot of these courses, and each course knows which plan it belongs to. Each line of this type also has its own **Type**, which is just a way of saying whether the course is compulsory, elective, or something else, and is identified by a `Code`. Additionally, each course in the plan is tied to a specific **Course Session**, which tells you exactly when and where the class is held, with a `Time` and a `Week_Day`.

The `Course Session` is obviously part of a larger **Teaching**, which is the actual subject. A single lesson can have several sessions, such as the Monday lesson and the Wednesday lesson. Each course, in turn, belongs to a **Department**, which is the larger structure such as "Engineering" or "Letters", each with its own name and code. Finally, each course is also linked to a specific **Room** and **Location**, which would be the campus or building in a certain `City`. So, it seems like a course is pretty fixed as to where it takes place.
