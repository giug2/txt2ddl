The data model describes a library management system divided into six main entities. The AUTHOR entity is identified by a CODE (PK) attribute and includes the fields SURNAME, NAME and DATE OF BIRTH. The ORGANIZATION entity is defined by a CODE (PK) and a NAME. The BOOK entity is uniquely identified by its ISBN (PK) and contains the TITLE attribute. The PUBLISHER is characterized by a CODE (PK) and a NAME. Each physical instance of a book is represented by the COPY entity, with an INVENTORY number as the primary key (PK) and a PURCHASE DATE. Finally, the USER entity is defined by a CODE (PK) and the SURNAME, NAME and PHONE attributes. The interconnections are defined by the following relationships: a relationship of cardinality N:1 links AUTHOR to ORGANIZATION; an N:N relationship exists between AUTHOR and BOOK; a N:1 relationship connects BOOK to PUBLISHER. Each BOOK can have multiple physical instances, as described by the 1:N relationship with the COPY entity. Loan management is modeled by an N:1 relationship between COPY and USER, enriched by the "return date" attribute to track the transaction.
