In practice, at the center of everything are the events, which would be the individual concerts or shows. Each event is a bit particular, to identify it you have to put a few things together: which tour it belongs to, where it takes place and a kind of repeat number, like if they do multiple dates in the same place. Obviously, each event also has its own specific name. These events are all part of a larger tour; a tour has its own code and name, and clearly a single tour brings together a lot of different events. Likewise, each event is held in a location, a place with its own identifier and name, and a place can host quite a few events over time. Then there is the artist, who has his own unique code and some generic data about him, and the important thing is that an artist can be connected to an infinite number of events, in short, he is the one who plays. Finally, there is something called the setlist, which is basically the list of songs from a concert, with its own code and a song title. What's a little strange is that these setlists seem to be linked directly to the locations, so a single place can have a lot of different setlists associated with it, perhaps to keep track of what was played there.
