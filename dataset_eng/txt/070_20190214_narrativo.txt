Welcome to the heart of our logistics system, a universe designed to track the journey of every package, from point of departure to arrival. At the center of this structure is the **Shipping**, the fundamental event that represents a single shipment, identified by a unique code. Each shipment is a story that begins with a protagonist: the **Sender**. Each sender, registered with his own code and name, can initiate countless shipments over time, but every single shipment originates from a single person or company.

The journey of each expedition has a precise destination, its **Destination**. Although a destination can be the arrival point for an infinite number of different shipments, each shipment is directed towards a single specific location, characterized by its own code and description. To manage the complexity of logistics flows, each destination is in turn grouped into a broader geographic or logistics **Category**, such as a continent or region. This means that a category can include multiple destinations, but each destination belongs to only one category, allowing for clear, hierarchical organization.

Furthermore, each shipment is characterized by a **Type** of service, which defines the methods and priorities, such as "standard" or "express". A service type can be applied to an unlimited number of shipments, but each shipment is classified according to a unique and specific type.

A shipment, however, is almost never a single item. It is a container, a whole. For this reason, each shipment can be made up of one or more **Packages**. Each package is inextricably linked to its shipment and is identified by a progressive number valid only within that shipment. Finally, to better manage storage and transport, each individual package is classified according to a specific **Size** (for example, "small", "medium", "bulky"). Of course, the same size category can be assigned to an indefinite number of different packages, providing an essential standard for the physical management of the goods.
