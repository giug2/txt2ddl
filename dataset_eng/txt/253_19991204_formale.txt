The data model describes a structure for managing information in the musical field, structured around the entities Artist, Song, Music Genre, Record Label and Album. The Artist entity is uniquely identified by an Artist Code (PK) and includes the Surname and Name attributes. The Song entity is defined by a Song Code (PK) and a Title. The Musical Genre is identified by a GenreCode (PK) and described by the Name attribute. The Record Label is identified by the Label Code (PK) and contains the Name and Location fields. The Album entity is defined by an AlbumCode (PK) and a Name. There is also the Publication entity, which is identified by a composite primary key made up of fk_song and fk_label and has the Year attribute. The relationships between entities establish that there is a many-to-many (N:N) relationship between Artist and Song. A one-to-many (1:N) relationship links Music Genre to Song, implying that each song belongs to a single genre. Finally, the Song, Album, and RecordLabel entities are linked to Release via distinct one-to-many (1:N) relationships.
