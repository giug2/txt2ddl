The data model describes the User and Order entities. The User entity represents the customer registry and is uniquely identified by the Email attribute, which acts as the primary key (PK). The additional attributes that characterize it are Surname, Name, Registration Date and Shipping Address. The Order entity represents a purchase transaction, uniquely identified by an Order ID (PK). A one-to-many (1:N) relationship is defined between User and Order: a single User can place multiple Orders, but each Order is associated with only one User. This association implies the presence, in the Order entity, of a foreign key that references the primary key of the User entity.
