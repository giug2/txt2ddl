The data model defines a system for managing projects and customers. Classification entities include `ClientSector`, identified by the primary key `CSett`; `ProjectStatus`, with key `CStatus`; `Department`, with `CRep` key and `Name` attribute; and `Priority`, with key `CPrio`. Each `Customer`, identified by `CCCustomer`, is associated with a unique `CustomerSector` in a one-to-many relationship. The central `Project` entity, identified by a unique `ID`, is linked to a single `Customer`, a `ProjectState` and a `Department`, establishing one-to-many relationships with each of these entities. Each project can have multiple `Tasks` associated with it. The `Task` entity is defined by a primary key composed of `NTask` and `fk_project`, which acts as a foreign key to the `Project` entity, ensuring that each task belongs to only one project. Finally, each `Task` is associated with a single `Priority` in a one-to-many relationship. The `CustomerSector`, `Customer`, `ProjectStatus` and `Priority` entities also contain a descriptive attribute called `Description`.
