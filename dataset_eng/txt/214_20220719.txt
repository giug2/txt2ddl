The system describes the management of software projects whose core is represented by the Projects entity. Each project is identified by a Project_ID, has a Name, a Description and is associated with a specific Team that develops it and a destination Platform. A single project can be divided into multiple development phases; the Sprint entity manages this information, linking each Project to a specific End_Date and Duration for each cycle. The backlog of each project consists of a set of Features to be implemented. The N:N relationship between Feature and Project establishes this link, indicating which features are included in a given project. Each Feature is in turn identified by an ID_Feature, described by a Name and requested by a Customer. The requester is a Customer entity, registered with Customer_ID, Last Name and First Name. Teams are defined by their Name and the Office they belong to. Similarly, Platforms are characterized by a Platform_ID and a Name. The workforce is managed by the Developers entity, where each individual is identified by a Serial Number and registered with Surname and Name. The hierarchical structure within a team is modeled through a recursive relationship on the Assignment entity, allowing you to define supervisory roles. The actual assignment of tasks is defined by the Assignment entity, which links a Developer to a Team and specifies the Technology for which he or she is responsible for that role. Finally, the Technologies entity catalogs the development tools, each with its own Technology_ID, a Name and a Category to which it belongs (e.g. language, database, framework).
