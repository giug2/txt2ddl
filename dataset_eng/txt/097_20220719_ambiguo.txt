So, basically here we have a few things that revolve around the world of concerts. There is the piece of music, which has its own title and a code to recognize it, and obviously each piece was written by a composer, who has his own name, surname and also a code. A composer may have written a lot of pieces, but a piece is always by only one author. These pieces then end up in concerts; a concert can have many different pieces in it, and the same piece can be played in a lot of concerts.

Each concert has a title and a description, and is identified by a code. This concert, intended as a program, is then performed in certain events, which are simply dates with a specific time. A single concert program can be replicated on multiple dates. Each concert is linked to a specific theatre, which has its name and a code, and is played by a certain orchestra, which has a name that identifies it and a venue. Of course, an orchestra or theater can host a lot of different concerts over time.

Speaking of the orchestra, it is made up of musicians, each with their own number, name and surname. It seems that each musician belongs to only one orchestra. Then there's something a little strange that we call "composition". This stuff, which only has one code, seems to be how orchestra musicians bond with the instruments they play. In fact, every musician and every orchestra is connected to these "compositions". In turn, each "composition" is linked to a specific instrument, which has a name, a type and a code. So, basically, through this "composition" we know who plays what in that orchestra. What's even more confusing is that a "composition" can be somehow connected to other "compositions", as if there were some kind of hierarchy or link between the roles, but it's not entirely clear how it works.
