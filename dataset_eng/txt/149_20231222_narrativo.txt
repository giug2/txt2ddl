This system was designed to orchestrate the complex and dynamic life of a university, mapping the entire academic and logistical structure, from the largest organizational entities down to the single detail of a lesson. At the top of this structure we find the UNIVERSITY, identified by a unique code and its name, which acts as the main container for all activities. Each university is a vibrant ecosystem, made up of numerous FACULTIES and Departments. Each FACULTY, in turn defined by a code and a name, represents a branch of the organization and has responsibility for the physical infrastructure; in fact, each CLASSROOM, the physical place where teaching takes place, belongs to one and only one faculty.

At the same time, academic life is organized into DEPARTMENTS, which also belong to a single university and identified by a code and a name. It is within a department that each ACADEMIC COURSE is created and managed. A course is uniquely defined not only by its specific code, but also by the department that delivers it, creating an indissoluble link between the training offer and its area of ​​scientific expertise.

Planning a course is a detailed process. First of all, a weekly timetable is established, associating each course with the LESSON DAYS on which it will be held; This flexible relationship allows a course to run on multiple days of the week and, of course, each day to host classes from multiple courses. Subsequently, each session of an academic course is assigned to a specific ROOM, and for this combination a precise Start Time and End Time are established, thus defining the official calendar.

Modern teaching often requires technological supports. For this, the system manages the necessary technical resources. Each academic course can be associated with a required TYPE OF EQUIPMENT, such as a projector or interactive whiteboard, defined by a code and name. These types are general categories. The system then tracks each individual physical EQUIPMENT, identified by a unique serial number, specifying which type it belongs to.

Finally, to move from planning to reality, the system records each ACTUAL LESSON. This entity captures what really happens: for a given course, on a specific DATE, the ACTUAL START and ACTUAL END times are recorded. In this way, it is possible to monitor the actual progress of the activities. To complete the picture, each actual lesson is connected to the specific EQUIPMENT, identified by its serial number, which was used on that occasion, ensuring complete traceability and efficient management of the university's material resources.
