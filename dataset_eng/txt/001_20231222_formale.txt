The data model describes an air traffic management system. The basic geographic entities are COUNTRY, identified by a CODE (PK), and CITY, identified by a CODE (PK); a nation can contain multiple cities in a one-to-many relationship. Each city can host multiple AIRPORT entities, each with its own CODE (PK). The AIRLINE entity is identified by a CODE (PK) and is associated with a COUNTRY to which it belongs. Aircraft specifications are managed by the AIRCRAFT TYPE entity, with CODE (PK), which defines a model, and by the AIRCRAFT entity, with CODE (PK), which represents a physical instance; There is a one-to-many relationship between a type and its aircraft. The flight concept is structured hierarchically. The FLIGHT entity represents a scheduled flight, identified by a composite primary key made up of NUMBER and fk_company, establishing a 1:N relationship with COMPANY. A flight is associated with an AIRCRAFT TYPE and two airports, one for departure and one for arrival. The associative entity FLIGHT ON THE DAY defines the weekly schedule, connecting a FLIGHT entity to a DAY entity (identified by NUMBER). The primary key of FLIGHT ON THE DAY is composed of fk_day and fk_flight and includes the attributes DEPARTURE AIRPORT and ARRIVAL AIRPORT. Finally, the ACTUAL FLIGHT entity represents the concrete instance of a flight on a specific date, identified by a composite key of DATE and fk_flight_of_day, creating a 1:N relationship with FLIGHT ON DAY. This entity records the ACTUAL DEPARTURE and ACTUAL ARRIVAL times and is associated with the specific AIRCRAFT that operated the route.
