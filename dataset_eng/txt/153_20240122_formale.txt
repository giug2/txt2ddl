The data model defines the structure of an academic information system, divided into four main entities: FACULTY, COURSE, PROFESSOR and STUDENT. Each entity is uniquely identified by a primary key (PK): `pk_facolta` for FACULTY, `pk_corso` for COURSE, `pk_professore` for PROFESSOR and `pk_studente` for STUDENT. The relationships between these entities are defined as follows. A FACULTY can offer multiple COURSES, while each COURSE belongs to only one FACULTY, establishing a one-to-many relationship. The relationship between STUDENT and COURSE is many-to-many, indicating that a student can enroll in multiple courses and a course can be attended by multiple students. Similarly, there is a many-to-many relationship between PROFESSOR and COURSE, so a professor can teach several courses and a course can be taught by multiple professors. Finally, a many-to-many relationship connects PROFESSOR and STUDENT, allowing a professor to mentor multiple students and a student to be mentored by multiple professors.
