So, basically, we have a system that manages things called Missions, which are kind of big objectives, each with its own number and name. Each Mission is divided into different Lines, which are like lines of activity; to recognize a Line, you need both its number and the number of the Mission to which it is attached. These Lines also have a name and, importantly, a budget. Then there are the Recommendations, which are suggestions with their own code and name, and these can be connected somewhat haphazardly to multiple Missions, and a Mission can follow several Recommendations.

Returning to the Lines, the money in their budget comes from various Sources, which are simply entities with a code and a name. A Line can take funds from multiple different Sources and a Source can finance many Lines; when this happens, the exact amount that passes from one party to the other is also noted. Each Line, in turn, contains a lot of concrete Projects, each with its own code and name, but each Project refers to one and only one Line.

Finally, the Ministries come into play, identified by a code and a name. The management of the Projects is a bit intertwined: for each Project there is a Ministry that acts as Coordinator, therefore it is the head. However, the same Project can also have many other Ministries participating, and in fact these Participants are kept track of. Thus, a Ministry can be the head of some projects and a mere participant in many others.
