The data model describes a system for managing airline flights. The basic geographic entities are `NATION` and `CITY`, each identified by a `CODE` and a `NAME`; a nation can contain multiple cities. Each city can host multiple `AIRPORT` entities, also defined by a `CODE` and a `NAME`. Similarly, the `COMPANY` entity is associated with a single `NATION` to which it belongs. The core of the system is the `FLIGHT` entity, identified by a composite primary key made up of the `NUMBER` of the flight and a foreign key that uniquely links it to a `COMPANY`. Each `FLIGHT` is characterized by two distinct relationships with `AIRPORT`: one to define the departure and one for the arrival, each with the attribute that specifies the scheduled time. The weekly frequency of a `FLIGHT` is managed via a many-to-many relationship with the `DAY` entity. The model distinguishes between `AIRCRAFT_TYPE`, which represents an aircraft category, and `AIRCRAFT`, which represents the specific physical instance; each `FLIGHT` is associated with an `AIRCRAFT_TYPE`. Finally, the `ACTUAL_FLIGHT` entity represents the executed instance of a `FLIGHT` on a specific `DATE`, forming a composite primary key. This entity records the actual departure and arrival times and connects, via a many-to-one relationship, to the specific `AIRCRAFT` that operated the route.
