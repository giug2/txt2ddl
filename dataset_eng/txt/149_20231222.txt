The system describes an academic management domain organized around several interconnected entities. The fundamental entities are the Universities, each uniquely identified by a code and characterized by a name. The Departments are connected to these, each defined by its own code, a name and a direct reference to the university it belongs to. The organizational structure is further detailed by the Faculties, also identified by a code and a name, and for each of which the university to which they belong is of interest. The faculties host the Classrooms, each specified by an identification code, a name and a link with the faculty in which it is located. The heart of the model is represented by the Academic Courses, understood as programmed cycles of lessons. A course is uniquely identified by the pair of attributes formed by the department that provides it and its course code. For each course, the room where the lesson begins and ends, the scheduled start and end times, and the type of teaching equipment designated for that course are recorded. It is important to note that the same course can be held on different days of the week, even with different times. To manage the technical specifications of teaching resources, there is an Equipment Types entity, which lists the various existing models using a code and a name. In addition to the types, the individual physical equipment is also of interest, each identified by a unique serial number and associated with a specific type of equipment. Finally, to track real activities, Actual Lessons are recorded. Each actual lesson represents a specific instance of a scheduled course, occurring on a specific date, identified by department, course code, and date, and records the actual start and end times. A fundamental constraint establishes that each actual lesson must use specific equipment, which must necessarily be of the type associated with the corresponding scheduled course.
