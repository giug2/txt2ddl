The data model defines a system for managing music festivals, structured around four main entities: Festival, Artist, Stage and Exhibition. The Festival entity is identified by a unique Code (PK) and has attributes such as Name, StartDate and EndDate. The Artist entity is defined by a unique Code (PK), a Musical Genre and the Training Year. Similarly, the Stage entity is characterized by a unique Code (PK), a Name and its Capacity. The Performance entity serves as an associative table and represents an artist's single performance at a given festival; its primary key is composite, made up of the foreign keys fk_festival and fk_artist, establishing that an artist can only perform once for each specific festival. This structure implements one-to-many relationships between Festival and Exhibition and between Artist and Exhibition. Finally, a one-to-many relationship connects Stage to Performance, indicating that each performance takes place on a single stage, while one stage can host multiple performances.
