So, here we have a whole system for keeping track of concerts and music stuff. In practice, there are musical pieces, each with its own title and code so as not to confuse them. Each piece, obviously, was written by a composer, who we identify with a code and his name. A composer may have created a lot of pieces, but each piece has only one dad, so to speak. These pieces are then played during concerts. A concert is a program, with its own title and description, and can include a lot of different pieces; on the other hand, the same piece can be performed again in many concerts. Each concert program is performed by one and only orchestra, which we recognize by its name and which has a venue and a conductor. Clearly, the same orchestra can play many different concert programs over time. Then there is the practical part: the event. An event is when a concert takes place on a certain date and at a certain time. The same program can be replicated in multiple events, such as Friday and Saturday evenings. Each evening takes place in a specific theatre, which has its own name and code, and obviously a theater can host an infinite number of events. The orchestra, then, is made up of people: the musicians, each with their own number, name and surname. To know who plays in which orchestra, there is a kind of register, the composition, which links a musician to an orchestra. The combination of an orchestra and a musician creates a unique entry on this list. Finally, for every musician who is part of the composition of an orchestra, we also know what instrument he plays. Each instrument has its own name and a code, so that specific musician in that given orchestra plays only one instrument, although obviously there can be many musicians who play the same type of instrument.
