So basically we have a system that tracks a few things about live music. There is a list of all the musical pieces, each with its own identification code and title, just to know what we are talking about. Each piece is connected to a musician, such as the composer, who in turn has his own code and name. A musician may have written a lot of pieces, but each piece is the work of only one of them. Then there are the concerts, which are the real events. Each concert has its own title and code to avoid confusion, plus a general description. The interesting thing is that a concert is practically a program, a set list of various pieces, and the same piece can be played in a lot of different concerts. Each concert program can be staged several times, and these are the reruns; to know which performance it is, you need to know what concert we are talking about and what date it was held. Obviously, every concert takes place in a room, which has a name, a code and we know how many people are in it. A hall, of course, can host quite a few different concerts over time. An orchestra always plays, identified by its name and with a conductor at the head. The same orchestra can do a lot of concerts. Inside the orchestras there are the musicians, the actual musicians, each with their own number, name and surname. The part that ties it all together is that we know who plays in which orchestra; a musician can be part of multiple orchestras and an orchestra is made up of many musicians. And importantly, for each orchestral player in a given orchestra, we also know what instrument he plays.
