So, basically here we're talking about how the whole thing is organised. There are our products, which are the main thing; each product has its own identification code, a name, a description of what it does and a support contact, as well as an indication of whether it is available or not. A single product can have a lot of different versions, such as colors or sizes, and each of these variants has its own code and obviously its own selling price. Then, to sell more, we have promotions: a promotion, which we identify with a name and a code, can be applied to a lot of different products, and at the same time a product can end up in multiple special offers at the same time.

All this merchandise is somewhere, that is, in our warehouses. Each warehouse is identified by a code and the name of the location. A specific product can be found in only one warehouse, it is not scattered around. These warehouses are supplied by various companies, suppliers, each with its own company name, a code and an idea of ​​the average cost they make. The slightly strange thing is that each supplier seems to be tied to only one warehouse.

Finally, there's the delivery part. We have shipping companies, couriers, each with its own name and ID. A courier handles shipments for quite a few warehouses, but each warehouse has only one reference courier. These couriers then move in certain geographical areas, which are simply areas with a name and their own code. The thing is quite free: a courier can cover many different areas and, vice versa, a single area can be served by more than one courier.
