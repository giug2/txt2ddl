So, in this system we have some stuff that goes around a university, more or less. There are Departments, each with its own identification code, name and location where they are located. Then there are the Faculties, these too with a code and a name to recognize them. Each department is linked in some way to a lot of degree courses, and to be honest even a degree course can be linked to multiple departments, so it's quite flexible. Same story for teachers: a professor, who we identify with his student number and whose name and surname we know, can hang out in multiple departments, and each department obviously has its own team of teachers. Speaking of degree courses, which also have a code and a name, each of these is made up of a list of teachings. A course, which is a specific subject with its own code and name, is only part of a specific degree course. And not only that, each teaching is also headed by one and only one Faculty, which in turn manages several; in fact, it seems that to identify a teaching you need to know both its code and which faculty it is linked to. Finally, we are interested in knowing something about the teachers. For each professor, we keep track of the city in which he was born and the city in which he resides. Of course, multiple teachers can be born or live in the same place, and each city has its own code to avoid confusion.
