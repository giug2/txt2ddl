The data model describes the structure for managing academic information, divided into four main entities. The City entity is identified by a unique Code (PK) and contains the Name and Country attributes. The University entity is defined by a Code (PK), a Name and a Foundation date. The Professor entity is identified by a Code (PK) and includes personal data such as Surname, Name and date of birth. The Qualification entity, identified by a Code (PK), describes the academic title through the Name attribute. Interconnections are defined by one-to-many relationships: a City can host multiple Universities, but each University is located in only one City. Similarly, a University employs multiple Professors, with each Professor belonging to a single University. Finally, each Professor is associated with a specific Qualification, which can be common to multiple professors.
