So, in practice there is a whole system for managing health information, starting from the people, the patients. We know the tax code of each patient, which is the important thing to avoid confusion, and then the usual things like name, surname and date of birth. When one of these patients is ill, he ends up hospitalized, and each hospitalization is linked to that specific person; obviously a person can have multiple hospitalizations in their life. Each hospitalization has its own identification code and it is known in which department it occurred.

These hospitalizations, then, do not take place in the air, but inside a hospital, so each hospitalization is connected to a specific structure. A hospital, which has its own code and name, obviously manages a lot of hospitalizations. In turn, hospitals are not entities in their own right, but report to an ASL, which is the larger organization that supervises them. A single ASL, with its code and name, can have quite a few hospitals under it, but a hospital only responds to one ASL.

Returning to hospitalization, this generates costs, so for each hospitalization one or more invoices may emerge, each with its number, date and amount to be paid. Finally, these invoices must be paid. For a single invoice there can be several payments, maybe one pays a little at a time. Each payment is a kind of transaction with its own ID, the date it was made and the amount paid, and it is always linked to a specific invoice, it is not paid at random.
