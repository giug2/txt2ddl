So, basically we have a system for organizing the university. There is the `Faculty`, which has its own identification code and name, nothing strange. Each faculty has quite a few `Degree Courses` under it, and each course belongs to only one faculty. You can recognize a 'Degree Course' by an acronym and the code of its faculty, a sort of double identifier, as well as having a full name. Then things get a little more complicated: a degree course is made up of many `Subjects`, and the same subject can be present in multiple courses. When you associate them, you also specify which `Course Year` that subject belongs to for that course. `Matter` itself is very simple, it just has a code and that's it. Each degree course also has its own `Lessons`, each with its own code, title, timetable and duration. Finally, there is a list of `Days of the Week`, with code and name. The strangest connection is between lessons and days: a lesson can be scheduled over multiple days and obviously there are multiple lessons in one day. When you create this pairing, you also note the `Room Capacity` for that specific lesson and day combination.
